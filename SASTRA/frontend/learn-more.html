How SASTRA Works

SASTRA is a static application security testing (SAST) platform that combines rule-based scanning, abstract syntax tree analysis, and machine learning to identify vulnerabilities in your source code — before they reach production.

Scan Pipeline — Step by Step
1
Upload your code
Submit a single source file (.py, .java, .cpp, .c, .h) or a ZIP archive containing an entire project. SASTRA supports Python, Java, and C/C++ out of the box.
2
AST & regex scanning
For Python, SASTRA parses the code into an Abstract Syntax Tree and applies context-aware checks — flagging dangerous patterns only when they are genuinely unsafe (e.g. SQL built by string concatenation, subprocess with shell=True). For Java and C++, 462+ regex rules map directly to CWE IDs.
3
ML enrichment
Every finding passes through a Random Forest classifier trained on 14,000+ real-world vulnerability pairs. The model predicts a vulnerability probability score and a severity level (Critical / High / Medium / Low). Findings scoring below the confidence threshold are flagged as likely false positives.
4
Remediation Queue
The top 5 most impactful issues are surfaced in a prioritized Remediation Queue, ranked by severity, rule confidence, and estimated fix effort so your team knows where to start.
5
Fix generation & export
Click "Fix now" on any finding to generate a secured replacement using an LLM. Export a full PDF security report with all findings, code snippets, CWE references, severity breakdown, and remediation guidance — ready to share with clients or your security team.

ML Engine & False Positive Reduction

Static analysis tools are notorious for generating false positives — flagging secure code as vulnerable. SASTRA addresses this with a two-model ML pipeline:

  • Severity Predictor — A Random Forest classifier trained on real vulnerability data assigns a severity level based on the CWE type and code context.
  • FP Classifier — A separate binary classifier, trained on 14,442 samples (7,300 vulnerable + 7,142 safe/patched code pairs), estimates whether each finding is a genuine vulnerability or a false positive. Findings with a combined ML + rule confidence score below the threshold are flagged automatically.
  • CWE Enrichment — The feature text fed to the ML model is enriched with CWE descriptions, consequence keywords, and detection method tokens from the official CWE dataset — giving the TF-IDF vectorizer semantic signal beyond just the raw code.
Supported Languages & File Types
  • Python — .py files. Full AST-based analysis with context-aware checks for injection, path traversal, code execution, insecure deserialization, and more.
  • Java — .java files. Regex + AST scanning covering SQL injection, command injection, insecure deserialization, weak crypto, and hardcoded credentials.
  • C / C++ — .c, .cpp, .cc, .cxx, .h, .hpp files. Comprehensive buffer overflow, use-after-free, integer overflow, command injection, and unsafe function detection.
  • ZIP archives — Upload a multi-file project as a ZIP (up to 25 MB, 250 files). SASTRA scans every supported file and aggregates findings into a single unified report.
PDF Reports & Scan History

Every scan is stored in your personal scan history. You can revisit previous scans, compare results over time, and export a professional PDF security report at any point.

The PDF includes: executive summary, severity breakdown chart, per-finding details with code snippets, CWE and OWASP references, ML confidence scores, recommended remediations, and a false positive summary.
